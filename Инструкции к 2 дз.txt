Инструкции:
1) Файл payments необходимо вставить в папку с файлами по контрактам и анкетам.
2) Установить среду разработки PyCharm https://www.jetbrains.com/pycharm/
3) В настройках установить библиотеки:numpy, pandas, pip, pyodbc, python-dateutil, pytz, scikit-learn, scipy, 
setuptool, sklearn, tabulate, xlrd.
Пример: setting -> project interpreter -> плюсик -> в поле поиска xlrd -> install
4) Откройте в  PyCharm 3 файла: first task, second task, general. В general содержится общий код.
5) Запустите сначала first task,затем second task.
6) После запуска второго скрипта пользователю будет предложено выбрать горизонт риска (то есть пишем кол-во дней).
7) Далее выбираем категориальную переменную пола (то есть пишем gender) для расчета woe и iv.

Результаты:
1) К базе данных по контрактам и анкетам добавлена таблицу платежей.
2) Соединили Python с БД по ODBC
3) Для каждого id вычислим наличие просрочки свыше 90 дней. Если сумма назначенных платежей за три месяца превосходит
выплаченную сумму за этот же период, и эта ситуация сохранится в четвертом месяце, то с этого момента будем считать этот кредит дефолтным.
4) При вводе пользователем горизонта риска выводится таблица заемщиков с пометкой дефолт/не дефолт на выбранном горизонте с момента выдачи кредита.
5) По выбранной характеристике из анкетных данных заемщика строятся показатели (таблица и картинка): WOE и IV.
